Browser Automation

The AI drives a real Electron browser window. It can open pages, navigate, run arbitrary JavaScript in the page context, and read back results and console output.

What the AI can do

  1. Open a browser — a separate Electron window dedicated to AI-driven automation (kept apart from the in-app webview tab to avoid visual clutter).
  2. Navigate to URLs — load any address.
  3. Execute JavaScript — run arbitrary JS in the page context and get the result back.
  4. Read console logs — retrieve console output captured from the page.
  5. Save files — dialog-based file saving.

The Browser API server

Browser control is a plain HTTP server run by the Electron main process, so any process can drive it — not just the LLM backend. The port is OS-assigned per instance (so multiple instances don't collide) and registered with the Overmind; read an instance's browser_port from GET http://127.0.0.1:9200/arms.

MethodPathPurpose
POST/browser/openOpen the AI browser window
POST/browser/navigateNavigate to {url}
POST/browser/executeRun {code} in the page; returns result
GET/browser/urlCurrent URL
GET/browser/logsCaptured console log buffer
POST/browser/logs/clearClear the log buffer

The console log buffer is populated by browser-preload.js, which intercepts console.* in the browser window and forwards entries to the main process.

Scripting it directly

Because the API is just HTTP, you can script the live browser from any tool while the app is running — for example with Python's urllib:

import json, urllib.request

PORT = 0  # read from GET http://127.0.0.1:9200/arms -> browser_port

def post(path, body):
    req = urllib.request.Request(
        f"http://127.0.0.1:{PORT}{path}",
        data=json.dumps(body).encode(),
        headers={"Content-Type": "application/json"},
    )
    return json.load(urllib.request.urlopen(req))

post("/browser/open", {})
post("/browser/navigate", {"url": "https://example.com"})
print(post("/browser/execute", {"code": "document.title"}))

Heads up

/browser/execute runs arbitrary JavaScript in the page context. Only point it at pages and code you trust.

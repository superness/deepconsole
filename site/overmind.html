Multi-instance & the Overmind

DeepConsole can run as several independent instances — separate Electron processes, each with its own window, sessions, and AI-controlled browser — that are aware of one another through the Overmind.

Shared backend, not killed

Instances don't fight over ports. main.js probe-then-spawns: the first instance to launch starts the shared LLM backend (8000) and the Overmind (9200); the rest detect them via /health and attach. Each instance gets its own OS-assigned browser API port and a persisted arm identity (arm.json in Electron userData).

The Overmind watches and creates space — it never directs

The Overmind is a standalone passive coordinator on port 9200 (a FastAPI service). It holds:

Presence roster

A live list of which instances are up, what they're focused on, and their browser ports.

Blackboard

Self-organizing work items any idle instance can atomically claim — pull, not push. No assignment, no sequencing.

Peer-asks

Routes questions from one instance to another and returns the answer.

The Ghost

When a board item is marked done, records a learning into the shared knowledge base.

All changes fan out live over GET /stream (SSE) — no polling. The design rule: if a change ever turns the Overmind into an assigner or scheduler, that's a smell — initiative belongs in the instances.

Autonomous worker mode

Any instance can be flipped into Autonomous mode from the Overmind tab. A DOM-free state machine (renderer/autonomous.js, unit-tested via npm test) then watches the live board feed and, while idle:

  1. Atomically claims one open item at a time.
  2. Runs it to completion in the instance's visible chat (the full abuddi agent).
  3. Marks the item Done with a summarized result — so the Ghost records a learning.
  4. Releases it on failure.

One job at a time (a single in-flight guard); disabling mid-job finishes the current job, then stops. A worker only ever takes work it claims — nothing is pushed to it, preserving the Overmind's pull-not-push contract.

Running a fleet

The Fleet Manager (fleet_manager.pyw) is a small native GUI to launch, monitor, and stop instances: launch N arms each with its own Electron profile and workspace, see the live roster, watch backend/Overmind health, and read the board summary (open / claimed / done).

Testing

The Overmind has its own test suite under overmind/tests/, and the autonomous state machine is tested with npm test (node --test).

Troubleshooting

The app pings http://127.0.0.1:8000/health on startup until the backend responds. Most setup issues are the backend not coming up — start there.

The app hangs on "waiting for backend"

The Electron UI is up but the abuddi backend on port 8000 never reported healthy. Check, in order:

  • Is abuddi a sibling of deepconsole? The app spawns the backend from ../abuddi. Both repos must share a parent folder.
  • Are the Python deps installed? cd abuddi && pip install -r requirements.txt.
  • Does the backend start on its own? Run it manually to see the real error: 
    cd abuddi
python -m uvicorn server:app --host 127.0.0.1 --port 8000
  • Run the app with DevTools — npm run dev — and check the logs for [LLM] lines from the spawned backend.

"DEEPSEEK_API_KEY not configured" / 500 on chat

The backend started but has no key. Create abuddi/.env with:

DEEPSEEK_API_KEY=sk-your-key-here

Get a key from platform.deepseek.com, then restart the app. A 401 from DeepSeek instead means the key is present but invalid or out of credit.

Port 8000 already in use

This is usually fine: DeepConsole probe-then-spawns, so a second instance detects the backend already running on 8000 and attaches to it rather than starting its own. If a different program holds 8000, stop it (or change the backend port) so the health check can succeed.

The backend restarts don't pick up code changes

The port-8000 uvicorn process runs without --reload. After editing backend .py files, fully restart the process (or the app) for changes to take effect.

npm install or Electron fails to launch

  • Make sure you're on Node 18+ (tested with v24): node -v.
  • Delete node_modules and reinstall if a partial install left it inconsistent.
  • On Linux, Electron needs the usual desktop libraries (e.g. libgtk, libnss3) present.

Multi-instance: the Overmind tab is empty

The Overmind runs on port 9200, started by the first instance to launch. Confirm it's up:

curl http://127.0.0.1:9200/health
curl http://127.0.0.1:9200/arms

If nothing answers, no instance has started it yet — launch one, or check that 9200 isn't blocked by another process.

Still stuck?

Open an issue on GitHub with the [LLM] log output from npm run dev and your Node / Python versions.
